Durant cette étape, pour les pipelines, je trouve que c’était presque la même chose que le Lab 03. Donc je n’ai pas rencontré de très grandes difficultés au début. La partie qui m’a un peu ralentie, c’était l’utilisation des conditions if dans GitHub Actions. Par exemple, il fallait utiliser if pour faire un comportement différent selon que c’était un push, une pull_request ou un workflow_dispatch. C’était nouveau pour moi de mettre des conditions comme if: github.event_name == 'push' ou pour vérifier certains paramètres. J’ai dû prendre un peu de temps pour bien comprendre comment ça fonctionne.

Au début, j’ai eu aussi un peu de mal à comprendre la partie `workflow_dispatch`, surtout comment fonctionnent les paramètres que l’on peut définir et utiliser dans le workflow. La différence entre les environnements QA et PROD n’était pas très claire pour moi au départ. Je ne comprenais pas bien comment la valeur choisie pouvait être utilisée dans le job de déploiement. Mais une fois que j’ai compris comment récupérer ces valeurs avec `github.event.inputs`, le fonctionnement est devenu beaucoup plus clair. L’expression conditionnelle `&& ||` était aussi nouvelle pour moi, mais elle a été assez facile à comprendre et à utiliser après quelques essais.

La vraie difficulté principale était avec GitLab. Je connaissais GitLab de nom, mais je ne l’avais jamais vraiment utilisé. J’ai trouvé que GitHub Actions et GitLab CI sont assez différents, même si le principe est le même. Au début, j’ai fait comme sur GitHub, j’ai essayé d’organiser le pipeline de la même manière, alors que GitLab fonctionne avec stages et rules, et le fichier doit être placé à la racine du projet.

J’ai aussi galéré pour copier mon répertoire GitHub vers GitLab. Je ne savais pas qu’il fallait vérifier les remotes avec la commande git remote -v pour voir vers quelle adresse le projet pointe. Au début, je ne comprenais pas pourquoi ça ne poussait pas vers GitLab. Ensuite, même quand j’ai réussi à ajouter GitLab comme remote et à faire un git push, GitLab a refusé pour la branche main parce qu’elle était protégée par défaut. Je ne savais pas que main était protégé automatiquement. Du coup, j’ai dû aller dans les paramètres de GitLab pour enlever temporairement la protection de la branche avant de pouvoir faire le push correctement.
